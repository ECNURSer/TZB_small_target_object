# YOLO26-OBB 实验计划

## 原则

1. 固定 fold、随机种子和输入尺寸；记录模型规模、batch 和训练长度差异。
2. 当前已有 s/m 的 fold 0 结果，下一轮重新训练 n；不运行五折。
3. 训练验证集指标和独立 test 指标分开记录。
4. n/s/m 使用相同的 balanced focal 分类 loss；box、DFL、angle loss 和模型结构保持官方实现。

## 训练停止与权重保存

- n/s/m 使用 `patience: 200`；设置为 0 可关闭早停。
- `last.pt`：每次保存时覆盖，始终表示最新训练状态，可用于断点续训。
- `best.pt`：验证 fitness 创造新高时覆盖，只保留当前最优模型。
- n/s 使用 `save_period: 50`，m 已完成实验使用 `save_period: 100`。

|---|---|---:|---:|---:|---|
| EXP-01A | yolo26n-obb.pt | 0 | 640 | 144 | 历史结果已删除，不参与比较 |
| EXP-01B | yolo26n-obb.pt | 0 | 1024 | 64 | 待运行 700 epoch 优化实验 |

n/s 使用相同 batch 64，m 历史实验使用 batch 72，因此当前结果属于实用最优配置对比，不是严格的单变量架构消融。若 26n 发生 OOM，将 batch 降到 48；不要让 TaskAlignedAssigner 回退到 CPU 后继续长时间训练。

## 第二阶段

直接使用 fold 0 验证集结果比较 n/s/m。单折实验不能量化不同数据划分带来的波动，因此结果只用于本次固定划分下的模型选择。

| 模型 | mAP50 | mAP50-95 | F1 | 推理 ms/图 | 参数量 M |
|---|---:|---:|---:|---:|---:|
| YOLO26s 500 epoch 基线 | 0.5357 | 0.3938 | 0.5507 | 3.29 | 9.76 |
| YOLO26m 500 epoch | 0.5994 | 0.4463 | 0.6143 | 10.61 | 21.21 |

当前 m 精度领先，s 在速度和模型规模上占优。等待 EXP-01B 完成后再按最终判定规则选择模型。

## 第三阶段

在独立 test 有真实标签的前提下，只对已经确定的最终方案使用 `evaluate_test.py` 做最终评估。test 指标不得再用于调整模型、超参数或数据增强。后续若需要模型融合，应另写 OBB 融合与提交脚本，并单独记录融合指标。

## 最终判定规则

1. 主指标：fold 0 验证 mAP50-95，以及最终独立 test mAP50-95。
2. 类别表现：逐类别 AP，避免总体指标掩盖弱类别。
3. 辅助精度：mAP50、Precision、Recall 和 F1。
4. 部署成本：同卡、同输入尺寸和同 batch 下的单图推理耗时与参数量。
5. 当两个架构的 mAP50-95 差小于 0.005 时，优先选择更小、更快的模型。
