# YOLO26-OBB 比赛优化计划

## 评分目标

评分规范见 `/home/dihan/评分规范.pdf`。比赛按旋转框 IoU 阈值 0.3 统计 TP、FP、FN，并计算：

```text
Precision = TP / (TP + FP)
Recall    = TP / (TP + FN)
F1        = 2TP / (2TP + FP + FN)
```

排行榜首先比较历史最高 F1；仅当 F1 相同时，运行时间更短的方案排名更高。因此：

1. 比赛 `F1@OBB-IoU0.3` 是唯一主指标。
2. mAP50、mAP50-95 和 Ultralytics F1 只作为训练诊断指标，不能代替比赛 F1。
3. 不为了速度牺牲可确认的 F1；只有 F1 相同或差异小于评估误差时才比较速度。
4. 所有阈值和后处理只在验证集调整，独立 test 不用于反复调参。

## 当前基线

n/s/m 均已使用 fold 0 验证集和 `best.pt` 完成单卡复评。统一设置为 `imgsz=1024`、预测最低置信度 `0.05`、`max_det=1000`、评分 IoU `0.3`。物理 GPU 为 7 号卡，每次只用一张卡。YOLO26 OBB 为 NMS-free，命令中的 `nms_iou` 不影响这些预测。

当前排名为 m > s > n。m 比 s 高 0.0255 F1，比 n 高 0.0806 F1。表中推理时间是 Ultralytics 报告的单卡模型 inference 时间，不是比赛并列时所需的端到端严格基准。

对当前最优 m 模型完成后处理搜索后，主方案更新为 `imgsz=1280、max_det=600、conf=0.32508`，F1=0.755425。逐类别 conf 可达 0.756307，但提升仅 0.000882，暂不作稳健主方案。

主方案冻结后已在独立 test 评估一次：固定 conf=0.325080，TP=35990、FP=12122、FN=11559，Precision=0.74805、Recall=0.75690、F1=0.752449。本轮不再根据 test 结果调整参数。

评分 PDF 未明确匹配细节，当前本地评分器的可复现假设是：同类别匹配、预测按置信度降序、每个 GT 最多匹配一次、polygon IoU >= 0.3 记为 TP。在官方评测代码或补充规则确认前，上述分数是对齐已知规则的本地基线，不宣称与官方评分器完全等价。

验证集共 1703 张图、86298 个目标，平均每图 50.7 个，单图最多 596 个。7 张图超过 300 个目标，若沿用默认 `max_det=300`，理论上最多会因数量上限额外产生 784 个 FN。实测 `max_det=600` 与 1000 几乎同分且 600 略高，因此当前主方案使用 600。

## 实验约束

- 固定 fold 0、seed 42、数据转换规则和类别映射。
- 每次实验只改变一个主要因素，并记录完整配置、权重、比赛 F1 和端到端耗时。
- n/s/m 均使用官方 OBB 预训练权重初始化。
- 新训练器已启用梯度、AMP、模型缓冲区和 EMA 的 NaN/Inf 检查与 DDP 回滚。
- 周期权重必须保留，因为 Ultralytics `best.pt` 不一定是比赛 F1 最优权重。
- 单次改动若 `F1@0.3` 提升小于 0.002，默认视为收益不足，除非多个验证子集结果一致。
