# YOLO26-OBB 比赛优化计划

## 评分目标

评分规范见 `/home/dihan/评分规范.pdf`。比赛按旋转框 IoU 阈值 0.3 统计 TP、FP、FN，并计算：

```text
Precision = TP / (TP + FP)
Recall    = TP / (TP + FN)
F1        = 2TP / (2TP + FP + FN)
```

排行榜首先比较历史最高 F1；仅当 F1 相同时，运行时间更短的方案排名更高。因此：

1. 比赛 `F1@OBB-IoU0.3` 是唯一主指标。
2. mAP50、mAP50-95 和 Ultralytics F1 只作为训练诊断指标，不能代替比赛 F1。
3. 不为了速度牺牲可确认的 F1；只有 F1 相同或差异小于评估误差时才比较速度。
4. 所有阈值和后处理只在验证集调整，独立 test 不用于反复调参。

## 当前基线

当前 s/m 数值来自 fold 0 验证集、`imgsz=1024` 的 `best.pt` 复评：

| 模型 | Epoch | mAP50 | mAP50-95 | Ultralytics F1 | 推理 ms/图 | 参数量 M | 比赛 F1@0.3 |
|---|---:|---:|---:|---:|---:|---:|---:|
| YOLO26s | 500 | 0.5357 | 0.3938 | 0.5507 | 3.29 | 9.76 | 待评估 |
| YOLO26m | 500 | 0.5994 | 0.4463 | 0.6143 | 10.61 | 21.21 | 待评估 |
| YOLO26n | 700 | - | - | - | - | - | 待训练 |

上述 F1 是由 Ultralytics Precision/Recall 推导的值，不是评分规范要求的 `F1@IoU0.3`。当前只能判断 m 的常规检测指标领先，不能据此确定比赛最终排名。

验证集共 1703 张图、86298 个目标，平均每图 50.7 个，单图最多 596 个。7 张图超过 300 个目标，若沿用默认 `max_det=300`，理论上最多会因数量上限额外产生 784 个 FN。因此正式评估和提交至少使用 `max_det=1000`。

## 实验约束

- 固定 fold 0、seed 42、数据转换规则和类别映射。
- 每次实验只改变一个主要因素，并记录完整配置、权重、比赛 F1 和端到端耗时。
- n/s/m 均使用官方 OBB 预训练权重初始化。
- 新训练器已启用梯度、AMP、模型缓冲区和 EMA 的 NaN/Inf 检查与 DDP 回滚。
- 周期权重必须保留，因为 Ultralytics `best.pt` 不一定是比赛 F1 最优权重。
- 单次改动若 `F1@0.3` 提升小于 0.002，默认视为收益不足，除非多个验证子集结果一致。
